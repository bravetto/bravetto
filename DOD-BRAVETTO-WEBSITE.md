## testing

### contract validation (`npm run test:sales`)

- `scripts/validate-sales.js` -- 56-line node.js validator
- required patterns: `#voice-overlay`, `#abe-trigger`, `.chip`, `<elevenlabs-convai`, elevenlabs cdn, fixed positioning

- `tests/sales.spec.js` -- playwright browser test

## drift risks

1. **tailwind cdn breaking change** -- all pages depend on cdn.tailwindcss.com. major version bump could break styling across 33+ pages simultaneously.
2. **elevenlabs convai widget api change** -- voice integration on sales.html and abevoice.html depends on external widget. breaking change = broken CTA.
3. **p5.js cdn deprecation** -- fixed at 1.9.0 via cdnjs. version pinned but cdn could go offline.
4. **google fonts availability** -- preconnect to googleapis.com. outage degrades typography gracefully (falls back to georgia/system-ui).
5. **vercel rewrite sprawl** -- 35 rewrites manually maintained. adding pages without updating vercel.json = 404s.
6. **sitemap drift** -- sitemap.xml manually maintained. new pages added without sitemap update = seo gap.
7. **submodule staleness** -- scream-machine submodule may diverge. `update-submodules.sh --no-pull` mitigates but doesn't prevent.
8. **aec content aging** -- sales proposal references specific dates (feb 20 signing), numbers (6,214 parts). stale if not updated with real engagement.
9. **no automated accessibility testing** -- lighthouse/axe not wired into ci. accessibility regressions silent.
10. **single test coverage** -- only sales.html has e2e tests. other 32 pages have zero automated validation.
